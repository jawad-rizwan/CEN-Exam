# CEN 800 Quiz App

A tiny, local browser quiz app for practicing **CEN 800: Law and Ethics in the
Engineering Practice** questions. No frameworks, no backend. You run a small
local server, open a page in your browser, pick a deck, and go.

## Quick start

Already have Python? Three commands, then open a URL.

```bash
# 1. install the one library needed by the extractor
pip install pymupdf

# 2. build the question bank from the PDFs (run this once)
python scripts/extract_questions.py

# 3. start a local web server in the project folder
python -m http.server 8000
```

Then open this in your browser:

<http://localhost:8000/app/>

On Windows, if `python` is not recognized, use `py` instead:

```bash
py -m pip install pymupdf
py scripts/extract_questions.py
py -m http.server 8000
```

Press `Ctrl+C` in the terminal to stop the server when you are done.

---

## What this app is

- One question at a time.
- Multiple choice and true/false.
- Questions are shuffled every session so you don't memorize order.
- Immediate feedback and the correct answer after each response.
- Score and progress shown the whole time.
- End-of-quiz summary with a "retry missed questions" button.
- Deck selection: Lecture 7 & 8, Lecture 9 & 10, Lecture 11 & 12, Extra, or All
  Randomized.

All questions are pre-extracted once into `data/question_bank.json`. The app
reads that file directly. The PDFs are not reparsed at runtime.

## Step-by-step for first-time users

These instructions assume you are new to this project and want to run it on
your own computer.

### 1. Install Python (if you don't already have it)

- Windows: install from <https://www.python.org/downloads/windows/>. During
  install, tick **"Add python.exe to PATH"**.
- macOS: install from <https://www.python.org/downloads/macos/> or run
  `brew install python` if you use Homebrew.
- Linux: use your distro's package manager, e.g. `sudo apt install python3`.

Verify it worked:

```bash
python --version
```

If that fails on Windows, try:

```bash
py --version
```

Whichever one works is the command you'll use below.

### 2. Download the project

Either clone it with Git:

```bash
git clone <repo-url>
cd CEN-Exam
```

Or on GitHub click the green **Code** button -> **Download ZIP**, then unzip
the folder and open a terminal inside it.

### 3. Install the one Python library the extractor needs

```bash
pip install pymupdf
```

On Windows, if `pip` isn't recognized:

```bash
py -m pip install pymupdf
```

### 4. Run the extraction script (once)

This reads the PDFs in `Questions/` and writes `data/question_bank.json`.

```bash
python scripts/extract_questions.py
```

Windows alternative:

```bash
py scripts/extract_questions.py
```

You should see a summary with how many questions were kept per deck. If the
file `data/question_bank.json` now exists, you're good.

You only need to re-run this if the PDFs change.

### 5. Start the local web server

Still in the project folder:

```bash
python -m http.server 8000
```

Windows alternative:

```bash
py -m http.server 8000
```

Leave that terminal running.

### 6. Open the app

Open this URL in your browser:

<http://localhost:8000/app/>

Pick a deck, click **Start Quiz**, and start practicing.

To stop, go back to the terminal and press `Ctrl+C`.

## Troubleshooting

**"python is not recognized"** (Windows)
Use `py` in place of `python` everywhere, e.g. `py scripts/extract_questions.py`
and `py -m http.server 8000`. If even `py` isn't found, reinstall Python and
tick "Add python.exe to PATH" during setup.

**`pip install` fails with a permission error**
Try:

```bash
python -m pip install --user pymupdf
```

**The app says "Failed to load questions"**
Make sure:
- you started the server from the project root (the folder that contains
  `app/`, `data/`, `scripts/`, and `Questions/`),
- the file `data/question_bank.json` exists (re-run step 4 if not),
- you opened `http://localhost:8000/app/` and not `file://...`. Browsers block
  file-based `fetch` requests, which is why you need the tiny local server.

**Port 8000 is in use**
Pick any free port, e.g.:

```bash
python -m http.server 8765
```

and open <http://localhost:8765/app/>.

## What the extraction script does

- Reads every PDF in `Questions/`.
- Pulls out numbered questions, choices, and the printed `Answer:` letter.
- Classifies each question as `multiple_choice` or `true_false`.
- Attaches scenario context (e.g. `Foundation and Ground Inc.`) to the
  questions it belongs to, based on ranges like `Questions 41 to 46`, and
  stamps each question with a `group` id so the app can keep scenario
  questions together when shuffling.
- Detects "scenario referenced but body is missing" placeholders (e.g. the
  blocks in *Lectures 7 & 8 Part 3* labelled `Questions 26-30`, `31-35`,
  `36-40`, `41-46`) and drops those questions entirely. The script prints a
  line like `Skipped 21 questions due to missing scenario context`.
- Skips anything malformed or missing an answer and prints why.
- Writes the cleaned list to `data/question_bank.json`.

It does not try to guess answers. If an answer is missing from the source,
that question is skipped, not invented.
